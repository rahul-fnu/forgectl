# Field Notes: Research B

## Snapshot
- Focus area: usage patterns in mid-size engineering teams.
- Collection method: lightweight interviews and ticket log sampling.
- Date window: January to February 2026.

## Key Observations
1. Teams move faster when runbooks are short and example-driven.
2. Ownership is clearer when alerts map to explicit service maintainers.
3. Weekly review rituals reduce repeated incident classes.

## Working Hypotheses
- Better onboarding docs reduce first-month support load.
- Small automation scripts outperform large one-time process rewrites.

## Next Actions
- Validate findings with one additional team.
- Quantify time-to-resolution changes after runbook updates.
