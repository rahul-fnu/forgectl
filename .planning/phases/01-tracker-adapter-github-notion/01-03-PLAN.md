    - postComment: POST /v1/comments with parent page_id and rich_text content
    - updateState: PATCH /v1/pages/{page_id} updating the status select property
    - updateLabels: PATCH /v1/pages/{page_id} updating the labels multi_select property
    - Throttle: all API calls go through throttle that ensures max 3 requests/second (tracks last request timestamps, delays if needed)
    - Property extraction: title -> title[0].plain_text, rich_text -> rich_text[].plain_text joined, select -> select.name, multi_select -> multi_select[].name, people -> people[].name, relation -> relation[].id
    - Rich text to markdown: converts Notion rich_text annotations (bold, italic, code, strikethrough) to markdown syntax
    - Page URL: https://notion.so/{page.id with dashes removed}
    - All requests include Authorization: Bearer {token} and Notion-Version: 2022-06-28 headers
  </behavior>
  <action>
    Create `src/tracker/notion.ts`:

    1. **Internal state:** Store resolved token, database_id, property_map (Record<string, string> mapping TrackerIssue field names to Notion property names), active_states, terminal_states, label config. Also: lastPollTime (string|null for delta polling), requestTimestamps (number[] for throttle).

    2. **Helper: `throttle()`** — Tracks timestamps of last 3 requests. Before each request, check if 3 requests happened within the last 1000ms. If so, delay until the oldest is > 1000ms ago. Simple implementation: store array of timestamps, shift old ones, sleep if needed.

    3. **Helper: `notionFetch(method, path, body?)`** — Calls throttle(), then fetch(`https://api.notion.com${path}`) with headers: Authorization: Bearer {token}, Notion-Version: 2022-06-28, Content-Type: application/json. On 429: read Retry-After header, wait, retry once. On non-2xx: throw with status and error body.

    4. **Helper: `extractProperty(page, notionPropName, propType)`** — Extracts value from a Notion page property based on type. Handles: title, rich_text, select, multi_select, people, url, relation, created_time, last_edited_time. Returns appropriate JS value.

    5. **Helper: `richTextToMarkdown(richTextArray)`** — Converts Notion rich_text array to markdown string. Each item has `text.content` and optional `annotations` (bold, italic, code, strikethrough, underline). Apply markdown syntax: **bold**, *italic*, `code`, ~~strikethrough~~. Join all items.

    6. **Helper: `normalizePage(page, propertyMap)`** — Maps Notion page to TrackerIssue using the property_map config. For each TrackerIssue field, look up the Notion property name from propertyMap, then extract the value. Default property_map if not configured: { title: "Name", status: "Status", priority: "Priority", labels: "Tags", assignees: "Assignee", description: "Description" }. id = page.id, identifier = page.id (first 8 chars), url = `https://notion.so/${page.id.replace(/-/g, "")}`, created_at = page.created_time, updated_at = page.last_edited_time.

    7. **`fetchCandidateIssues()`** — POST to /v1/databases/{database_id}/query. Build filter: if lastPollTime set, filter by last_edited_time > lastPollTime. Also filter by status property in active_states if adapter knows the status property name. Sort by last_edited_time descending. Paginate via start_cursor. Normalize all pages. Update lastPollTime to current ISO string.

    8. **`fetchIssueStatesByIds(ids)`** — For each id, GET /v1/pages/{id}, extract status property value. Return Map.

    9. **`fetchIssuesByStates(states)`** — Query database with filter: status property select equals each state (use OR compound filter). Paginate and normalize.

    10. **Write operations:**
        - postComment: POST /v1/comments with `{ parent: { page_id }, rich_text: [{ text: { content: body } }] }`
        - updateState: PATCH /v1/pages/{page_id} with `{ properties: { [statusPropName]: { select: { name: state } } } }`
        - updateLabels: GET current page to read existing labels, compute new multi_select array (add new, remove specified), PATCH /v1/pages/{page_id}

    11. **Factory:** Export `createNotionAdapter(config: TrackerConfig): TrackerAdapter` — validates config.database_id is present, resolves token, sets up property_map with defaults merged with config overrides.
