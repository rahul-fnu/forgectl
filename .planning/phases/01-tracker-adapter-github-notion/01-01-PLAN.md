    - TrackerConfigSchema validates kind:"github" requires repo field

    - TrackerConfigSchema defaults: active_states ["open"], terminal_states ["closed"]
    - ConfigSchema.tracker is optional (existing configs without tracker still parse)
    - TrackerConfigSchema includes label config fields: in_progress_label, done_label (optional strings)

       - Use `.superRefine()` to validate: if kind==="github", repo must be present; if kind==="notion", database_id must be present
       - Add `tracker: TrackerConfigSchema.optional()` to ConfigSchema
       - Export TrackerConfigSchema
