  <action>
    Create `src/workspace/manager.ts`:

    Export `WorkspaceInfo` interface: `{ path: string; identifier: string; created: boolean; }`

    Export `WorkspaceManager` class:
    - Constructor takes `config: z.infer<typeof WorkspaceConfigSchema>` and a logger (use project's Logger type from `src/logging/logger.ts`). Resolve root path (replace leading `~` with `os.homedir()`). Parse `hook_timeout` to milliseconds.
    - `async ensureWorkspace(identifier: string): Promise<WorkspaceInfo>` -- Sanitize identifier, build path, assert containment, mkdir with recursive:true, check if dir was freshly created (stat ctime vs now, or use a try/catch pattern with mkdir). If created, run after_create hook (let errors propagate). Return WorkspaceInfo.
    - `async runBeforeHook(identifier: string): Promise<void>` -- Sanitize, build path, assert containment, run before_run hook if configured. Let errors propagate.
    - `async runAfterHook(identifier: string): Promise<void>` -- Same but wrap in try/catch, log and ignore failures per user decision.
    - `async removeWorkspace(identifier: string): Promise<void>` -- Sanitize, build path, assert containment, run before_remove hook (catch and log failures), log deletion, rm with recursive+force.
    - `async cleanupTerminalWorkspaces(terminalIdentifiers: string[]): Promise<void>` -- Read root dir entries, build Set of sanitized terminal identifiers, remove matching ones. Handle missing root gracefully (return).
    - `getWorkspacePath(identifier: string): string` -- Sanitize, resolve, assert containment, return path. Helper used internally and exposed for orchestrator cwd validation (R3.3).

    For created detection: Use a two-step approach -- try `stat` first. If ENOENT, `mkdir` then `created=true`. If stat succeeds, `created=false`. This avoids the TOCTOU issue mentioned in research (mkdir recursive is idempotent but doesn't tell you if it created).
