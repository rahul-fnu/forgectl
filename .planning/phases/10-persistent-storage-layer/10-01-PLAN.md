<task type="auto" tdd="true">
  <name>Task 1: Install dependencies, create database singleton and schema</name>
  <files>package.json, tsup.config.ts, drizzle.config.ts, src/config/schema.ts, src/storage/database.ts, src/storage/schema.ts, test/unit/storage-database.test.ts</files>
  <behavior>
    - Test: createDatabase() with temp path creates a .db file at that path
    - Test: createDatabase() enables WAL journal mode (pragma query returns "wal")
    - Test: createDatabase() enables foreign keys (pragma query returns 1)
    - Test: createDatabase() sets busy_timeout to 5000
    - Test: createDatabase() with no path uses default ~/.forgectl/forgectl.db logic
    - Test: closeDatabase() closes the connection without error
    - Test: Drizzle instance has schema attached (can reference runs table)
  </behavior>
  <action>
    1. Install dependencies:
       ```bash
       npm install drizzle-orm better-sqlite3
       npm install -D drizzle-kit @types/better-sqlite3
       ```

    2. Update `tsup.config.ts`:
       - Add `external: ["better-sqlite3"]` to the config (native module cannot be bundled)
       - Add `cpSync("drizzle", "dist/drizzle", { recursive: true })` to onSuccess (ship migrations with dist)
       - Wrap the drizzle copy in a try/catch since the folder may not exist during early dev

    3. Create `drizzle.config.ts` at project root:
       ```typescript
       import { defineConfig } from "drizzle-kit";
       export default defineConfig({
         dialect: "sqlite",
         schema: "./src/storage/schema.ts",
         out: "./drizzle",
       });
       ```

    4. Create `src/storage/schema.ts` with Drizzle table definitions:
       - `runs` table: id (text PK), task (text not null), workflow (text nullable), status (text not null default "queued"), options (text JSON), submittedAt (text not null), startedAt (text nullable), completedAt (text nullable), result (text JSON), error (text nullable)
       - `pipelineRuns` table: id (text PK), pipelineDefinition (text JSON not null), status (text not null default "running"), nodeStates (text JSON), startedAt (text not null), completedAt (text nullable)
       - Use `sqliteTable` from `drizzle-orm/sqlite-core`
       - Column names in SQL use snake_case (e.g., `submitted_at`), TypeScript property names use camelCase

    5. Create `src/storage/database.ts`:
       - Export `createDatabase(dbPath?: string)` that:
         a. Resolves path to `dbPath ?? join(process.env.HOME || "/tmp", ".forgectl", "forgectl.db")`
         b. Creates parent directory with `mkdirSync(dirname(resolvedPath), { recursive: true })`
         c. Creates `new Database(resolvedPath)` from better-sqlite3
         d. Sets pragmas: `journal_mode = WAL`, `foreign_keys = ON`, `busy_timeout = 5000`
         e. Returns `drizzle(sqlite, { schema })`
       - Export type `AppDatabase = ReturnType<typeof createDatabase>`
       - Export `closeDatabase(db: AppDatabase)` that calls the underlying better-sqlite3 close. Access the raw client via the `$client` property on the drizzle instance (which is the Database instance from better-sqlite3).

    6. Add `storage` field to `ConfigSchema` in `src/config/schema.ts`:
       ```typescript
       storage: z.object({
         db_path: z.string().default("~/.forgectl/forgectl.db"),
       }).default({}),
       ```

    7. Write tests in `test/unit/storage-database.test.ts` using temp directories (mkdtempSync). Each test creates a database in a temp dir, verifies behavior, then cleans up. Use `db.$client.pragma("journal_mode")` to verify WAL mode.
  </action>
  <verify>
    <automated>FORGECTL_SKIP_DOCKER=true npx vitest run test/unit/storage-database.test.ts</automated>
  </verify>
  <done>
    - drizzle-orm and better-sqlite3 in package.json dependencies
    - drizzle-kit and @types/better-sqlite3 in devDependencies
    - tsup.config.ts externalizes better-sqlite3 and copies drizzle/ to dist/
    - drizzle.config.ts exists at project root
    - src/storage/schema.ts exports runs and pipelineRuns table definitions
    - src/storage/database.ts exports createDatabase, closeDatabase, AppDatabase
    - src/config/schema.ts has storage.db_path field
    - All database tests pass
  </done>
</task>
