  <name>Task 2: Worker lifecycle (buildOrchestratedRunPlan + executeWorker)</name>
  <files>src/orchestrator/worker.ts, test/unit/orchestrator-worker.test.ts</files>
  <behavior>
    - buildOrchestratedRunPlan sets input.sources[0] to workspace path
    - buildOrchestratedRunPlan sets input.mode to "repo"
    - buildOrchestratedRunPlan renders prompt template with issue data
    - buildOrchestratedRunPlan generates unique runId
    - executeWorker calls workspaceManager.ensureWorkspace with issue identifier
    - executeWorker calls workspaceManager.runBeforeHook before agent invocation
    - executeWorker calls workspaceManager.runAfterHook after completion
    - executeWorker creates CleanupContext with empty tempDirs (workspace persists)
    - executeWorker returns WorkerResult with agentResult and comment
  </behavior>
  <action>
Create `src/orchestrator/worker.ts` with:

1. `buildOrchestratedRunPlan(issue: TrackerIssue, config: ForgectlConfig, workspacePath: string, promptTemplate: string, attempt: number): RunPlan`
   - Generate runId with `crypto.randomUUID()`
   - Render prompt using `renderPromptTemplate(promptTemplate, buildTemplateVars(issue))` — add `attempt` to the template vars
   - Set `input.sources = [workspacePath]`, `input.mode = "repo"`, `input.mountPath = "/workspace"`
   - Pull agent settings from config.agent
   - Pull container settings from config (image from workflow or config.container.image)
   - Set `noCleanup = false` (container gets destroyed, but tempDirs is empty so workspace persists)
   - Set `output.mode = "git"`, `output.hostDir = workspacePath`
   - Pull validation steps from workflow config if present, otherwise empty
   - Set `orchestration.mode = "single"`

2. `WorkerResult` interface: `{ agentResult: AgentResult; comment: string; executionResult: ExecutionResult }`

3. `executeWorker(issue, config, workspaceManager, promptTemplate, attempt, logger, onActivity)`:
   - Call `workspaceManager.ensureWorkspace(issue.identifier)` to get workspace path
   - Call `workspaceManager.runBeforeHook(issue.identifier)` — if throws, return failure result
   - Build RunPlan via `buildOrchestratedRunPlan`
   - Create CleanupContext with empty `tempDirs` array (workspace must NOT be cleaned up)
   - Call `prepareExecution(plan, logger, cleanup)` to get container, binds, env
   - Create agent session with `createAgentSession()`, passing `onActivity` callback
   - Invoke agent session with rendered prompt
   - Optionally run validation loop if validation steps configured
   - Build structured comment via `buildResultComment()`
   - Call `workspaceManager.runAfterHook(issue.identifier)` (catch and log errors)
   - Call `cleanupRun(cleanup)` to destroy container
   - Return WorkerResult

Use mocks in tests for prepareExecution, createAgentSession, WorkspaceManager, and cleanupRun. Test the orchestration flow without real Docker.
