  <name>Task 1: Orchestrator class with startup recovery and graceful shutdown</name>
  <files>src/orchestrator/index.ts, test/unit/orchestrator-startup.test.ts</files>
  <behavior>
    - Orchestrator.start() calls startupRecovery then starts scheduler
    - startupRecovery fetches terminal-state issues and cleans their workspaces
    - startupRecovery failure logs warning but does not prevent orchestrator from starting
    - Orchestrator.stop() sets stopped flag, clears all retry timers
    - Orchestrator.stop() closes all running sessions with drain timeout
    - Orchestrator.stop() force-kills remaining agents after drain_timeout_ms
    - Orchestrator.stop() releases all claims and removes in_progress labels
    - Orchestrator.stop() resolves cleanly even if label removal fails
    - Orchestrator.isRunning() returns true after start, false after stop
  </behavior>
  <action>
Create `src/orchestrator/index.ts` with:

```typescript
export class Orchestrator {
  private state: OrchestratorState;
  private slotManager: SlotManager;
  private tracker: TrackerAdapter;
  private workspaceManager: WorkspaceManager;
  private config: ForgectlConfig;
  private promptTemplate: string;
  private logger: Logger;
  private stopScheduler: (() => void) | null = null;
  private running = false;

  constructor(opts: {
    tracker: TrackerAdapter;
    workspaceManager: WorkspaceManager;
    config: ForgectlConfig;
    promptTemplate: string;
    logger: Logger;
  })
```

Methods:

1. `async start(): Promise<void>`
   - Create fresh state via `createState()`
   - Create SlotManager with `config.orchestrator.max_concurrent_agents`
   - Run `startupRecovery()` (catch and log errors — don't block start)
   - Start scheduler: `this.stopScheduler = startScheduler({ state, tracker, workspaceManager, slotManager, config, promptTemplate, logger })`
   - Set `this.running = true`
   - Log: "Orchestrator started (max={max}, poll={interval}ms)"

2. `private async startupRecovery(): Promise<void>`
   - Fetch terminal-state issues: `await this.tracker.fetchIssuesByStates(this.config.tracker.terminal_states)`
   - For each terminal issue, call `this.workspaceManager.remove(issue.identifier).catch(err => logger.warn(...))`
   - Log count: "Startup recovery: cleaned N terminal workspaces"

3. `async stop(): Promise<void>`
   - Set `this.running = false`
   - Stop scheduler: `this.stopScheduler?.()`
   - Clear all retry timers: `clearAllRetries(this.state)`
   - Drain running workers (Promise.race with drain_timeout_ms):
     a. Close all sessions: `Promise.allSettled([...state.running.values()].map(w => w.session.close()))`
     b. Race against `setTimeout(resolve, config.orchestrator.drain_timeout_ms)`
   - Force-kill remaining: iterate running Map, call `cleanupRun(worker.cleanup).catch(() => {})`
   - Release all claims: iterate claimed Set, call `tracker.updateLabels(id, [], [config.orchestrator.in_progress_label]).catch(() => {})`
   - Clear state: `state.claimed.clear(); state.running.clear(); state.retryTimers.clear(); state.retryAttempts.clear()`
   - Log: "Orchestrator stopped"

4. `isRunning(): boolean` — returns `this.running`

5. `getState(): OrchestratorState` — returns current state (for API/dashboard access in Phase 6)

Write tests with mocked TrackerAdapter, WorkspaceManager, and AgentSession. Use vi.useFakeTimers() for drain timeout tests.
