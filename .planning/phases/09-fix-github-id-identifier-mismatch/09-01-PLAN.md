<objective>
Fix the cross-phase wiring bug where the GitHub adapter sets `id: String(ghIssue.id)` (internal numeric ID like "123456") instead of `id: String(ghIssue.number)` (issue number like "42"), causing 404s on all orchestrator mutation API calls.

Purpose: Without this fix, the entire E2E flow (GitHub issue -> dispatch -> agent -> comment -> close) fails silently on every tracker mutation. This is the last gap closure for v1.0.
Output: Fixed production code, updated test fixtures, new cross-phase integration test, full test suite green.
</objective>
