  <behavior>
    - EventRepository.insert() persists an event row; findByRunId() returns it with deserialized JSON data
    - EventRepository.findByRunIdAndType() filters by both runId and type
    - EventRepository.insert() with null data stores null, not "null" string
    - SnapshotRepository.insert() persists a snapshot row; findByRunId() returns all snapshots for a run
    - SnapshotRepository.latest() returns the most recent snapshot for a run (by id desc)
    - SnapshotRepository.latest() returns undefined when no snapshots exist
    - Both repos follow Phase 10 pattern: synchronous methods, JSON.stringify/parse in repo layer
  </behavior>
  <action>
    1. Add `runEvents` and `runSnapshots` table definitions to `src/storage/schema.ts`:
       - `runEvents`: id (integer, autoIncrement PK), runId (text, notNull), type (text, notNull), timestamp (text, notNull), data (text, nullable) -- JSON serialized
       - `runSnapshots`: id (integer, autoIncrement PK), runId (text, notNull), stepName (text, notNull), timestamp (text, notNull), state (text, notNull) -- JSON serialized
       - Use `integer("id").primaryKey({ autoIncrement: true })` from `drizzle-orm/sqlite-core`
       - Import `integer` alongside existing `text` import

    2. Run `npx drizzle-kit generate` to create the migration SQL file in `drizzle/`

    3. Create `src/storage/repositories/events.ts` following the runs.ts pattern exactly:

       - `EventInsertParams`: runId, type, timestamp, data? (unknown)
       - `EventRepository` interface: insert(params), findByRunId(runId), findByRunIdAndType(runId, type)
       - `createEventRepository(db: AppDatabase)`: factory function
       - `deserializeRow`: JSON.parse data if not null
       - insert: JSON.stringify data if not undefined/null

       - `SnapshotRepository` interface: insert(params), findByRunId(runId), latest(runId)
       - `createSnapshotRepository(db: AppDatabase)`: factory function
       - latest: query with eq(runId) + orderBy desc(id) + limit 1, return first or undefined

    5. Write tests in `test/unit/storage-events.test.ts` and `test/unit/storage-snapshots.test.ts`:
