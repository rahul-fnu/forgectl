    - Watcher debounces: multiple rapid changes result in single reload
    - Watcher keeps last good config when reload produces invalid YAML
    - Watcher keeps last good config when reload produces invalid schema
    - Watcher calls onWarning with error message on invalid reload
    - Watcher.stop() stops watching (AbortController)
    - Watcher.getLastGoodConfig() returns most recent valid config
  </behavior>
  <action>
    1. Create `src/workflow/watcher.ts`:
       - `WorkflowFileWatcher` class with:
         - `constructor(options: { debounceMs?: number })` — default debounceMs 300
         - `start(filePath: string, callbacks: { onReload: (config: ValidatedWorkflowFile) => void; onWarning: (message: string) => void }): Promise<void>` — uses `fs/promises` `watch()` with AbortController for cancellation
         - `stop(): void` — aborts the watcher, clears any pending debounce timer
         - `getLastGoodConfig(): ValidatedWorkflowFile | null` — returns the last successfully parsed config
         - Internal: debounce timer (clearTimeout/setTimeout pattern), lastGoodConfig reference, AbortController
         - On file change: debounce, then call `loadWorkflowFile(filePath)`. On success, update lastGoodConfig and call onReload. On error (parse or validation), keep lastGoodConfig unchanged, call onWarning with error message.
         - Per D4: onWarning should be called so the daemon can surface via both log and SSE events (the watcher itself just calls the callback; the daemon hooks it up to logger + SSE)
