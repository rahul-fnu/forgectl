**3. executeLoopNode() private method — add as a new method on PipelineExecutor:**

The method must follow this exact contract (per locked decisions in CONTEXT.md):

a. **Safety cap clamping:**
   - `const configuredMax = node.loop!.max_iterations ?? 10;` (default 10)
   - `const maxIterations = Math.min(configuredMax, GLOBAL_MAX_ITERATIONS);`
   - If `configuredMax > GLOBAL_MAX_ITERATIONS`, log a warning: `Loop "${node.id}": max_iterations ${configuredMax} clamped to ${GLOBAL_MAX_ITERATIONS}`

b. **Initialize loopState on nodeStates:**
   - Get the existing state from `this.nodeStates`
   - Set `state.status = "loop-iterating"`
   - Set `state.loopState = { currentIteration: 0, maxIterations, iterations: [] }`
   - Set `state.startedAt = new Date().toISOString()`

c. **Crash recovery check (LOOP-05):**
   - If `this.options.checkpointSourceRunId` is set, call `loadLoopCheckpoint(this.options.checkpointSourceRunId, node.id)`
   - If checkpoint exists: set `startIteration = lc.lastCompletedIteration + 1`, restore `state.loopState.iterations` from checkpoint, reconstruct progressive context file paths from iteration history

d. **Progressive context setup:**
   - Create a temp dir: `mkdtempSync(join(tmpdir(), "forgectl-loop-ctx-"))`
   - Maintain `progressiveContext: string[]` array of file paths

e. **Iteration loop — `for (let i = startIteration; i <= maxIterations; i++)`:**
   - Update `state.loopState.currentIteration = i`
   - Record `iterStartedAt = new Date().toISOString()`
   - Create a temporary node clone with modified task that includes progressive context header (all previous iteration outputs as numbered markdown sections)
   - Call `this.executeNode(clonedNode)` to run one iteration body
   - Read the result from `this.nodeStates.get(node.id)` — capture the iteration status ("completed" or "failed")
   - **IMPORTANT:** After `executeNode`, the state.status will be "completed" or "failed" — reset it back to "loop-iterating" for the next iteration
   - Push a `LoopIterationRecord` to `state.loopState.iterations`
   - Write iteration output to progressive context temp dir as `iteration-{NN}-output.md`
   - Add the file path to `progressiveContext` array
   - Save per-iteration checkpoint: `await saveLoopCheckpoint(this.pipelineRunId, node.id, i, state.loopState)`
   - Also save a regular checkpoint via `saveCheckpoint` so downstream hydration works
   - **Evaluate `until` expression:**
     - Build context: merge upstream dep statuses + loop-specific variables
     - `_status`: current iteration result ("completed" or "failed")
     - `_iteration`: current iteration number (1-based)
     - `_max_iterations`: the clamped maxIterations value
     - `_first_iteration`: `i === 1 ? 1 : 0` (numeric for filtrex)
     - Call `evaluateCondition(node.loop!.until, untilCtx as NodeStatusContext)` — note: the context has mixed types (strings + numbers) but evaluateCondition accepts Record<string, unknown> at runtime via its customProp handler
     - If `until` returns true: set `state.status = "completed"`, set `state.completedAt`, clean up temp dir, return
     - If `until` evaluation throws: set `state.status = "failed"`, set error, clean up, return (fatal — consistent with COND-06)

f. **Loop exhaustion (after loop ends without `until` becoming true):**
   - `state.status = "failed"`
   - `state.error = 'Loop "${node.id}" exhausted max_iterations (${maxIterations}) without "until" expression becoming true'`
   - `state.completedAt = new Date().toISOString()`

g. **Cleanup:** Wrap entire method in try/finally that removes the progressive context temp dir.

**4. Wire progressive context into executeNode:**

The simplest approach: instead of calling `this.executeNode(node)` directly for each iteration, create a shallow node clone where `node.context` includes the progressive context file paths. This way `executeNode` naturally passes them to the agent via `CLIOptions.context`. The node clone:
```typescript
const iterationNode = {
  ...node,
  context: [...(node.context ?? []), ...progressiveContext],
  task: node.task,  // task stays the same each iteration
};
```
