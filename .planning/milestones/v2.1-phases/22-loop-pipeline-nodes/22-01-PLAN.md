**checkpoint.ts additions:**

Add two new exported functions at the end of src/pipeline/checkpoint.ts. Import `LoopState` from `./types.js`.

1. `loopCheckpointPath` (private helper): returns `join(checkpointDir(pipelineRunId, nodeId), "loop-checkpoint.json")` — distinct from `checkpoint.json`.

2. `saveLoopCheckpoint(pipelineRunId, nodeId, lastCompletedIteration, loopState)`:
   - Creates checkpoint dir with `mkdirSync({ recursive: true })`
   - Writes `{ lastCompletedIteration, loopState }` as JSON to `loop-checkpoint.json`
   - Overwrites in-place (writeFileSync, not appendFileSync)

3. `loadLoopCheckpoint(pipelineRunId, nodeId)`:
   - Returns `{ lastCompletedIteration: number; loopState: LoopState } | null`
   - Returns `null` if file does not exist
   - Reads and parses JSON from `loop-checkpoint.json`
