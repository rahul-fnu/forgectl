    Create `src/skills/mount.ts`:

    1. Import `ContainerMounts` type from `../auth/mount.js`. Import `existsSync`, `readdirSync` from `node:fs`, `join` from `node:path`, `homedir` from `node:os`.

    2. Export `CREDENTIAL_DENY_LIST` as a `Set<string>` containing: `.credentials.json`, `credentials.json`, `auth.json`, `statsig`, `.env`, `token`, `api_key`.

    3. Export `validateNoCredentials(hostPath: string): void` — uses `readdirSync(hostPath, { recursive: true })` (Node 20+), splits each entry by `/` to get basename, throws if any basename matches deny list. Error message: `Skill mount security violation: credential file "${entry}" found in ${hostPath}. Remove it or use --no-skills to disable skill mounting.`

    4. Export `prepareSkillMounts(skills: string[], noSkills: boolean): { mounts: ContainerMounts; addDirFlags: string[] }`:
       - If `noSkills` or `skills.length === 0`, return empty mounts + empty flags.
       - Resolve `skillsBase = join(homedir(), ".claude", "skills")` and `agentsBase = join(homedir(), ".claude", "agents")`.
       - For each skill name: check `join(skillsBase, name)` — if exists, validateNoCredentials, push bind `"${hostPath}:/home/node/.claude/skills/${name}:ro"`, push addDirFlags `"--add-dir"` and `"/home/node/.claude/skills/${name}"` as SEPARATE entries.
       - Same for `join(agentsBase, name)` with container path `/home/node/.claude/agents/${name}`.
       - If missing, log at debug level and skip (use console.debug or import logger — keep simple, just skip silently for now).
       - Also check for host-level CLAUDE.md: `join(homedir(), "CLAUDE.md")` — if exists, add bind `"${path}:/home/node/CLAUDE.md:ro"` (no --add-dir needed for this).
       - Return `{ mounts: { binds, env: {}, cleanup: () => {} }, addDirFlags }`.

    Create `test/unit/skill-mount.test.ts`:

    Use `vi.mock("node:fs")` and `vi.mock("node:os")` to mock `existsSync`, `readdirSync`, and `homedir`. Test all behaviors listed above. Use `beforeEach` to reset mocks. For the credential tests, mock `readdirSync` to return arrays containing deny-listed filenames.

    IMPORTANT: `--add-dir` flag and its value MUST be separate array entries (not `"--add-dir /path"` as one string) because `buildShellCommand()` calls `shellEscape()` on each element individually.

    IMPORTANT: Do NOT import or use the project logger (would pull in too many dependencies for a utility module). Just skip silently — the CONTEXT.md says "log at debug level and skip silently" for missing dirs, which we implement by simply not adding them.
