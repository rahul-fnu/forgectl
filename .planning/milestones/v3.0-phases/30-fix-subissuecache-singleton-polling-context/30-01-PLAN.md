<objective>
Fix two wiring bugs that prevent sub-issue rollup and auto-close from firing for polling-originated issues (the primary production path).

Bug 1: `createGitHubAdapter()` creates a private SubIssueCache while `server.ts` creates a separate one — writes during `fetchCandidateIssues` go to Instance A, reads from scheduler go to Instance B (always empty). Fix: inject external cache into adapter factory.

Bug 2: `scheduler.ts` passes `undefined` as `githubContext` to `dispatchIssue`, so `triggerParentRollup` guard (`subIssueCache && githubContext`) always fails for polling-dispatched issues. Fix: add `githubContext` to `TickDeps` and wire it from Orchestrator via `setGitHubContext()`.

Purpose: Without these fixes, progress rollup and auto-close only work for webhook-triggered dispatches (minority path). Polling is the primary production path.
Output: 5 modified files, all existing tests green, new wiring tests passing.
</objective>
