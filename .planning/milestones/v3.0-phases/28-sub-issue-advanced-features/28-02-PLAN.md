  <action>
    **Modify `src/orchestrator/dispatcher.ts`:**

    1. Add imports at top: `import { upsertRollupComment, buildSubIssueProgressComment, allChildrenTerminal } from "../github/sub-issue-rollup.js"` and `import { SubIssueCache } from "../tracker/sub-issue-cache.js"`.

    2. Add optional `subIssueCache?: SubIssueCache` parameter to both `dispatchIssue()` and `executeWorkerAndHandle()` function signatures — after `githubContext`, keeping backward compat. Update the `void executeWorkerAndHandle(...)` call inside `dispatchIssue` to pass it through.

    3. Create a new helper function `triggerParentRollup`:
    ```typescript
    async function triggerParentRollup(
      childIssue: TrackerIssue,
      childSuccess: boolean,
      subIssueCache: SubIssueCache,
      tracker: TrackerAdapter,
      githubContext: GitHubContext,
      config: ForgectlConfig,
      logger: Logger,
    ): Promise<void>
    ```
    Logic:
    - Scan `subIssueCache.getAllEntries()` to find entry where `childIds.includes(childIssue.id)` -- this gives us the parent
    - If no parent found, return silently (not a sub-issue)
    - Update `entry.childStates.set(childIssue.id, "closed")` in-place for the just-completed child
    - Build `ChildStatus[]` from entry: for each childId, map state to ChildStatus state ("closed" -> "completed", "open" -> "pending"), construct URL as `https://github.com/{owner}/{repo}/issues/{childId}`, use childIssue.title for the just-completed child and `#${childId}` as fallback title for others
    - Call `buildSubIssueProgressComment(Number(entry.parentId), children)` to get comment body
    - Call `upsertRollupComment(githubContext.octokit as any, owner, repo, Number(entry.parentId), body)`
    - Check `allChildrenTerminal(entry.childStates, terminalStates)` where terminalStates = new Set from `config.tracker?.terminal_states ?? ["closed"]`
    - If all terminal: `tracker.updateLabels(entry.parentId, ["forge:synthesize"], []).catch(...)` with warn-and-swallow

    4. Insert `triggerParentRollup` call in `executeWorkerAndHandle` AFTER `executeWorker()` returns and BEFORE `tracker.postComment()` (~line 329). Wrap in try/catch that warns and swallows. Only call when `subIssueCache` and `githubContext` are both defined.

    5. Add synthesizer-gated close logic: In the success path (failureType === "continuation"), check if `issue.labels.includes("forge:synthesize")`. If yes:
       - Close the parent via `tracker.updateState(issue.id, "closed").catch(warn-swallow)`
       - Remove `forge:synthesize` label via `tracker.updateLabels(issue.id, [], ["forge:synthesize"]).catch(warn-swallow)`
       - Update the rollup comment with final summary (all completed)
       - Skip the normal `auto_close` and `done_label` logic for synthesizer runs (they handle their own close)
    In the failure path, if `issue.labels.includes("forge:synthesize")`:
       - Post error comment on parent explaining synthesizer failed
       - Do NOT close parent
