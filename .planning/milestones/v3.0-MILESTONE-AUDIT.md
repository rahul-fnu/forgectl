## Cross-Phase Integration (14/18 connections wired)

### Fully Wired (14 connections)

All sub-issue features (phases 25→28→29→30) are fully wired end-to-end:
- **SubIssueCache singleton:** server.ts creates one instance → shared between GitHub adapter and orchestrator
- **terminalIssueIds:** scheduler tick reads cache → builds terminal set → filterCandidates blocks parents
- **Progress rollup:** triggerParentRollup fires for both webhook and polling paths (githubContext now wired)
- **Auto-close:** allChildrenTerminal → forge:synthesize label → handleSynthesizerOutcome → close parent
- **Webhook invalidation:** cache.invalidate() on issues.edited events targets the shared singleton

Skill and team features (phases 26, 27) fully wired for CLI (`forgectl run`) path.

### Missing Connections (4 — non-critical)

**INT-01: Team config not mapped in orchestrated path**
- `mapFrontMatterToConfig()` does not handle `team` field
- Result: `team: {size: N}` in WORKFLOW.md parses correctly but never reaches `ForgectlConfig.team` in daemon mode
- Impact: Team features activate only via CLI `--team-size` flag, not via WORKFLOW.md in orchestrated runs
- Severity: Non-critical — TEAM-05 is a Should-priority requirement; all Must requirements work via CLI

**INT-02: Skills config not mapped in orchestrated path**
- `mapFrontMatterToConfig()` does not handle `skills` field
- `buildOrchestratedRunPlan()` hardcodes `skills: []`
- Result: WORKFLOW.md `skills: ["name"]` parses but skill mounts never activate in daemon mode
- Impact: Skill features activate only via CLI `forgectl run`, not orchestrated runs
- Severity: Non-critical — SKILL-04 is a Should-priority requirement; all Must requirements work via CLI

**INT-03: Duplicate GitHubContext type** (info — maintenance hazard)

**INT-04: Cycle detection uses console.warn** (info — not in structured logs)
