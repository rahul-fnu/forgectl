         runId: string;

  <behavior>
    - Test: :+1: reaction on bot comment calls approveRun with correct runId
    - Test: :-1: reaction on bot comment calls rejectRun with correct runId
    - Test: :rocket: reaction on issue body calls dispatch callback
    - Test: :arrows_counterclockwise: reaction on bot comment calls rerun callback
    - Test: reaction on arbitrary user comment (not bot, not issue body) is ignored
    - Test: reaction from user without write access is silently ignored
    - Test: :+1: reaction on comment with no associated run is silently ignored
    - Test: bot adds :eyes: reaction as acknowledgment when processing a valid reaction
  </behavior>
  <action>
    1. Create `src/github/reactions.ts`:

    2. Define ReactionDeps interface:
       ```typescript
       interface ReactionDeps {
         runRepo: RunRepository;
         onDispatch: (issue: TrackerIssue, octokit: Octokit, repo: RepoContext) => void;
         onRerun: (runId: string, octokit: Octokit, context: IssueContext) => Promise<void>;
       }
       ```

    3. REACTION_MAP constant mapping GitHub reaction content strings to actions:
       - "+1" -> "approve"
       - "-1" -> "reject"
       - "rocket" -> "trigger"
       - "confused" -> undefined (not our reaction, per user decisions it's :arrows_counterclockwise:)
       Note: Check GitHub API docs - :arrows_counterclockwise: emoji may not be a valid reaction type. If not available as a GitHub reaction, document this limitation and use a slash command for rerun instead. The actual GitHub reaction content values are: "+1", "-1", "laugh", "confused", "heart", "hooray", "rocket", "eyes".

    4. `handleReactionEvent(payload, octokit, deps: ReactionDeps): Promise<void>`:
       - Determine if reaction is on issue body (payload.comment is null/undefined, check event structure) or on a comment
       - For issue body reactions: only handle "rocket" (trigger run)
       - For comment reactions:
         a. Check if comment author is bot (comment.user.type === "Bot" or performed_via_github_app)
         b. If not bot comment, ignore
         c. Check hasWriteAccess for reactor
         d. If unauthorized, ignore silently
         e. Look up run via deps.runRepo.findByGithubCommentId(commentId)
         f. If no run found, ignore
         g. Map reaction to action:
            - "+1": call approveRun(deps.runRepo, run.id)
            - "-1": call rejectRun(deps.runRepo, run.id)
            - "rocket": call deps.onDispatch
            - For rerun: since :arrows_counterclockwise: is not a valid GitHub reaction, rerun is handled via `/forgectl rerun` slash command only
         h. Add :eyes: acknowledgment reaction

    5. Write tests with mock RunRepository and mock Octokit.
