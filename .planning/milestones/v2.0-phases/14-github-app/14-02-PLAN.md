    1. Update `src/github/webhooks.ts` — replace skeleton handlers with real logic:

    2. Define WebhookDeps interface for dependency injection:
       ```typescript
       interface WebhookDeps {
         triggerLabel: string;           // e.g., "forgectl"
         onDispatch: (issue: TrackerIssue, octokit: Octokit, repo: RepoContext) => void;
         onCommand: (cmd: ParsedCommand, octokit: Octokit, context: IssueContext, sender: string, commentId: number) => Promise<void>;
         runRepo: RunRepository;
       }
       ```

    3. `registerWebhookHandlers(app: App, deps: WebhookDeps): void`:
       - `issues.labeled` handler: Check if label.name matches deps.triggerLabel. If yes, convert payload to TrackerIssue format, call deps.onDispatch(). Skip if action !== "labeled".
       - `issues.opened` handler: Check if any label matches deps.triggerLabel. If yes, dispatch. Otherwise ignore.
       - `issue_comment.created` handler:
         a. Skip if comment author is bot (payload.comment.user.type === "Bot")
         b. Parse comment body with parseSlashCommand()
         c. If no command found, skip
         d. Check hasWriteAccess() for the comment author
         e. If unauthorized: add :x: reaction to comment, post error reply, return
         f. If authorized: add :eyes: reaction (acknowledgment), call deps.onCommand()

    4. Helper function `webhookPayloadToTrackerIssue(payload)` — converts GitHub webhook issue payload to TrackerIssue format matching existing tracker interface (id = issue number, identifier = "owner/repo#N", title, body, labels, etc.)

    5. Write tests using mock App webhooks and mock deps.
