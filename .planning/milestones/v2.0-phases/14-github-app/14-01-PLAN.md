       - Reads private key from disk via readFileSync(config.privateKeyPath, "utf-8")
       - Validates key starts with "-----BEGIN" (throw descriptive error if not)
       - Creates `new App({ appId, privateKey, webhooks: { secret: config.webhookSecret } })`
       - Exposes `app` property for webhook handler access
       - Exposes `getInstallationOctokit(installationId: number)` method
       - Export factory function `createGitHubAppService(config: GitHubAppConfig): GitHubAppService`

    3. Create `src/github/routes.ts` — Fastify webhook route:
       - Export `registerGitHubRoutes(fastify: FastifyInstance, appService: GitHubAppService): void`
       - Register a route-scoped content type parser for 'application/json' that preserves raw string body (parseAs: 'string')
         IMPORTANT: Use a Fastify encapsulated plugin (fastify.register with async function) to scope the content type parser to the webhook prefix only, preventing it from breaking JSON parsing on other routes
       - POST `/api/v1/github/webhook` handler:
         - Extract headers: x-github-delivery, x-github-event, x-hub-signature-256
         - Call `appService.app.webhooks.verifyAndReceive({ id, name, payload: rawBody, signature })`
         - Return 200 { ok: true } on success
         - Catch verification errors, return 401 { error: { code: "INVALID_SIGNATURE", message: "Webhook signature verification failed" } }
         - Catch other errors, return 500 with error envelope

    4. Create `src/github/webhooks.ts` — webhook handler registration scaffold:
       - Export `registerWebhookHandlers(app: App): void`
       - Register skeleton handlers on `app.webhooks.on()` for: "issues.labeled", "issues.opened", "issue_comment.created", "issue_comment.deleted"
       - Each handler logs the event type for now (actual logic in Plans 02-04)
       - Use typed payloads from @octokit/webhooks-types
