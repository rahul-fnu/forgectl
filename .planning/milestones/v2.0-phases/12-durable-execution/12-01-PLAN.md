    - releaseLock removes only the lock matching type+key+owner
    - releaseLock is a no-op if lock does not exist (idempotent)
    - releaseAllStaleLocks removes locks where daemonPid differs from current PID
    - releaseAllStaleLocks preserves locks owned by the current PID
  </behavior>

1. Create `src/durability/locks.ts` with three exported functions:

   `acquireLock(lockRepo: LockRepository, params: { lockType: "issue" | "workspace", lockKey: string, ownerId: string, daemonPid: number }): boolean`
   - Calls lockRepo.insert wrapped in try/catch
   - Returns true on success, false on unique constraint violation

   `releaseLock(lockRepo: LockRepository, lockType: string, lockKey: string, ownerId: string): void`
   - Calls lockRepo.deleteByOwner(lockType, lockKey, ownerId)

   `releaseAllStaleLocks(lockRepo: LockRepository, currentPid: number): number`
   - Finds all locks, filters those where daemonPid !== currentPid, deletes them
   - Returns count of released stale locks
