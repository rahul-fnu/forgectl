1. Create `src/agent/browser-use-session.ts` implementing AgentSession:
   - Constructor takes `(container: Docker.Container, agentOptions: AgentOptions, env: string[], sessionOptions?: AgentSessionOptions)`
   - Private fields: `alive = false`, `sidecarPort = 8765`, `onActivity` callback from sessionOptions
   - `invoke(prompt, options?)`:
     - If not alive: call `startSidecar()` then `waitForHealth()`, set alive = true
     - Call `runTask(prompt, timeout)` and return AgentResult
     - Call `onActivity?.()` when task starts
   - `isAlive()`: return this.alive
   - `close()`:
     - If alive, POST `http://localhost:${sidecarPort}/shutdown` via execInContainer (curl)
     - Set alive = false
   - `private startSidecar()`:
     - Use `execInContainer(container, ["sh", "-c", "python3 /usr/local/bin/browser-use-sidecar.py --port 8765 &"], { env: this.env })`
     - The `&` makes it a background process; execInContainer returns immediately
   - `private waitForHealth()`:
     - Poll `GET http://localhost:8765/health` via execInContainer with curl every 500ms
     - Parse response for `{"status": "ok"}`
     - Timeout after 30s (60 attempts) with descriptive error "Browser-use sidecar failed to start within 30s"

     - Parse JSON response. Map sidecar status to AgentStatus.
     - Return AgentResult with stdout = sidecar output, stderr = "", appropriate status, zero tokenUsage (browser-use doesn't report tokens), measured durationMs, turnCount = 1
