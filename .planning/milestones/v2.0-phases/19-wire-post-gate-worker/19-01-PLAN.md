<objective>
Wire the post-execution approval gate into the orchestrator worker path so that interactive and supervised autonomy levels work for webhook-triggered runs, not just CLI runs.

Purpose: Close the last v2.0 integration gap -- post-gate exists in single.ts (CLI) but not in worker.ts (orchestrator/webhook). Without this, governance config is silently ignored for webhook-dispatched runs.

Output: Updated worker.ts with post-gate, updated dispatcher.ts with run record insertion and governance passthrough, tests proving all paths.
</objective>

    if (governance?.autonomy && needsPostApproval(governance.autonomy) && governance.runRepo && governance.runId) {

        ? (agentResult.tokenUsage.input * 3 + agentResult.tokenUsage.output * 15) / 1_000_000
