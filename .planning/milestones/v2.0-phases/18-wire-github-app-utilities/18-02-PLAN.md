    - Test: at worker completion, completeCheckRun is called with success=true when agent completed and validation passed
    - Test: at worker completion, completeCheckRun is called with success=false when agent failed or validation failed

       - In `executeWorkerAndHandle()`, if headSha is available, store it in the run's options JSON via runRepo (if available).

    2. **Extend githubDeps in worker (from Plan 01):**
       - Add optional `headSha?: string` and `repoContext?: { owner: string; repo: string }` to the `githubDeps` parameter that Plan 01 added to `executeWorker()`.

         - Store returned checkRunId for later use (local variable).
       - Wrap in try/catch, log warning on failure.

    4. **Update check run after validation:**
       - After the validation loop (step 8), if checkRunId is set:
         - Call `updateCheckRun(octokit, owner, repo, checkRunId, "Validation in progress")`.
       - Wrap in try/catch.

         - Build summary using `buildCheckSummary(runResult)` where runResult is the RunResult already built for comment consolidation (from Plan 01's toRunResult).
         - Determine success: `agentResult.status === "completed"` AND (no validation OR validation passed).
         - Call `completeCheckRun(octokit, owner, repo, checkRunId, success, summary)`.
       - Wrap in try/catch.

    6. **Write tests:**
       - Mock octokit with checks.create and checks.update.
       - Test full lifecycle: create -> update -> complete.
       - Test skip conditions (no headSha, no octokit).
