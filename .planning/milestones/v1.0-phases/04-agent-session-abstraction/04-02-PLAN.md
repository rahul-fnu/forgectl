AgentSession contract (from CONTEXT.md locked decisions):
```typescript
type AgentStatus = "completed" | "failed" | "timeout" | "user_input_required";
interface TokenUsage { input: number; output: number; total: number; }
interface AgentResult {
  stdout: string; stderr: string; status: AgentStatus;
  tokenUsage: TokenUsage; durationMs: number; turnCount: number;
}
interface AgentSessionOptions { onActivity?: () => void; }
interface AgentSession {
  invoke(prompt: string, options?: { timeout?: number }): Promise<AgentResult>;
  isAlive(): boolean;
  close(): Promise<void>;
}
```

  <behavior>
    - Handshake: sends initialize request, receives result, sends initialized notification, sends thread/start, receives threadId
    - First invoke: spawns app-server + handshakes, then starts turn with full prompt
    - Turn lifecycle: sends turn/start, collects agent messages, resolves on turn/completed
    - Multi-turn: second invoke reuses existing threadId (no re-handshake)
    - Approval requests: auto-approved with { decision: "accept" } response
    - User-input-required: turn resolves with status "user_input_required"
    - Token usage: accumulated from thread/tokenUsage/updated notifications across turns
    - Activity callback: fires on any agent event (agent/message, turn/progress, etc.)
    - Timeout: turn rejects after timeout ms with status "timeout"
    - isAlive: false before first invoke, true after handshake, false after close
    - close: kills the exec stream, rejects any pending turn
    - invoke after close: throws "Session is closed"
    - JSONL line buffering: handles partial lines across chunks correctly
    - Malformed JSON lines are skipped (e.g., Rust log output on stderr)
  </behavior>
  <action>
  Create `src/agent/appserver-session.ts` with:

  **JsonLineReader helper class** (private, same file):
  - `buffer: string` initialized to ""
  - `feed(chunk: string): object[]` — appends to buffer, splits on "\n", parses complete lines, keeps partial trailing line. Skips empty lines and catches JSON parse errors.

  **AppServerSession class** implementing AgentSession:
  - Constructor: `(container: Docker.Container, agentOptions: AgentOptions, env: string[], sessionOptions?: AgentSessionOptions)`
  - Private state: `alive = false`, `threadId: string | null`, `turnCount = 0`, `totalTokens: TokenUsage` (zeros), `execStream: NodeJS.ReadWriteStream | null`, `lineReader: JsonLineReader`, `pendingRequests: Map<number, { resolve, reject }>`, `nextId = 1`, `onActivity` callback
  - Session state tracking: `state: "idle" | "invoking" | "closed"` to prevent double-invoke and invoke-after-close

  **spawn()**: Uses `container.exec()` with `Cmd: ["codex", "app-server"]`, `AttachStdin: true`, `AttachStdout: true`, `AttachStderr: true`. Starts with `hijack: true, stdin: true`. Sets up demuxed reading via docker.modem.demuxStream (stdout goes to lineReader, stderr logged). NOTE: The stream from hijack is bidirectional — write JSON-RPC to it, demux stdout for reading.

  Actually, since the bidirectional stream with demux is tricky (demuxStream expects separate writable targets), use this approach instead:
  - Use `container.exec({ Cmd: [...], AttachStdin: true, AttachStdout: true, AttachStderr: true, Tty: false })`
  - `exec.start({ hijack: true, stdin: true })` returns a raw stream
  - For writing: `stream.write(JSON.stringify(msg) + "\n")`
  - For reading: pipe through Docker's demux. Create two passthrough streams for stdout/stderr, use `docker.modem.demuxStream(stream, stdoutPassthrough, stderrPassthrough)`. Listen on stdoutPassthrough for data, feed to lineReader.
  - Store the raw stream for writing, stdoutPassthrough for reading.

  **sendRequest(method, params)**: Creates `{ method, id: this.nextId++, params }`, writes JSONL to stream, returns promise stored in pendingRequests map. No `jsonrpc` field (Codex omits it).

  **sendNotification(method, params)**: Creates `{ method, params }`, writes JSONL. No id, no response expected.

  **sendResponse(id, result)**: Creates `{ id, result }`, writes JSONL. Used for approval responses.

  **handleMessage(msg)**: Route by:
  - Has `id` + `result` and id is in pendingRequests → resolve the pending request
  - Has `id` + `error` → reject the pending request
  - `method === "item/commandExecution/requestApproval"` → auto-approve via sendResponse
  - `method === "thread/tokenUsage/updated"` → accumulate token usage
  - `method === "turn/completed"` → resolve current turn
  - `method === "turn/userInputRequired"` → resolve turn with user_input_required status
  - Any message with a method → fire onActivity callback

  **handshake()**: Send initialize request → receive result → send initialized notification → send thread/start request → store threadId. Apply read timeout (default 5s) on initialize response.

  **invoke(prompt, options?)**: Guard on state. On first call, spawn + handshake. Increment turnCount. Send `turn/start` with `{ threadId, input: [{ type: "text", text: prompt }] }`. Wait for turn completion (collect stdout from agent messages). Apply turn timeout (default from agentOptions.timeout or options.timeout). Return AgentResult.

  **isAlive()**: return `this.alive`
  **close()**: set state to closed, alive to false, destroy stream, reject pending requests
