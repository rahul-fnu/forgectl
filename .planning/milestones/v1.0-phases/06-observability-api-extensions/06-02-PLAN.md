  <behavior>
    GET /api/v1/state:
    - When orchestrator is configured and running: returns 200 with { status: "running", uptimeMs, running: [{issueId, identifier, startedAt, attempt, tokens}], retryQueue: [{issueId, identifier, nextRetryAt (from retryTimers), attempt}], slots: {active, max}, totals: {dispatched, completed, failed, tokens} }
    - When orchestrator is not configured: returns 503 with { error: { code: "NOT_CONFIGURED", message: "Orchestrator not running" } }

    GET /api/v1/issues/:identifier:
    - When issue found in running or metrics: returns 200 with { issue (TrackerIssue data), orchestratorState (running/completed/failed/retry_queued), session: {startedAt, lastActivityAt, attempt} or null, metrics: {totalAttempts, totalRuntime, tokens} }
    - When issue not found: returns 404 with { error: { code: "NOT_FOUND", message: "Issue not found" } }
    - When orchestrator not configured: returns 503

    POST /api/v1/refresh:
    - When orchestrator running: calls triggerTick(), returns 202 with { triggered: true }
    - When tick already in progress: returns 202 with { triggered: false, reason: "tick_in_progress" }
    - When orchestrator not configured: returns 503

    GET /api/v1/events:
    - SSE stream subscribing to runEvents "run:orchestrator" channel
    - On client disconnect: removes listener
    - Wraps write in try/catch (sink safety)

    Error envelope:
    - All error responses use { error: { code: string, message: string } } shape
  </behavior>

         - Get slots via `metrics.getSlotUtilization(state.running, slotManager)` — NOTE: SlotManager is inside Orchestrator. Add a `getSlotUtilization()` method on Orchestrator that delegates to `this.metrics.getSlotUtilization(this.state.running, this.slotManager)` (this avoids exposing slotManager directly)
