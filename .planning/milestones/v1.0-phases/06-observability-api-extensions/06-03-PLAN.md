<task type="auto">
  <name>Task 1: Orchestrator dashboard page component</name>
  <files>src/ui/index.html</files>
  <action>
    Read `src/ui/index.html` to understand the existing dashboard structure (navigation, routing, component patterns). Then add:

    1. Add "Orchestrator" navigation link alongside existing links (Dashboard, Pipelines, History, Settings, etc.) pointing to `#orchestrator`

    2. Add hash route case for "orchestrator" in the routing logic to render the OrchestratorPage component

    3. Create `OrchestratorPage` React function component with these sections:

       **a) Status Banner:**
       - Fetch from `/api/v1/state` on mount and every 10 seconds
       - Show "Running" (green) or "Stopped" (red/gray) status
       - Show uptime formatted as "Xh Ym" (from uptimeMs)
       - "Refresh Now" button: `POST /api/v1/refresh`, show brief "Triggered!" feedback, then re-fetch state

       **b) Slot Utilization:**
       - Visual bar showing active/max slots (e.g., "2/3 slots active")
       - Use Tailwind width percentage for the bar fill: `style={{ width: \`${(active/max)*100}%\` }}`
       - Color: green if active < max, yellow if active === max

       **c) Running Issues Table:**
       - Columns: Identifier, Started, Attempt, Tokens (total)
       - Each row clickable — toggles inline expansion
       - Expanded row shows: full token breakdown (input/output/total), started ISO timestamp, lastActivityAt if available
       - If no running issues, show "No agents running" message

       **d) Retry Queue:**
       - Columns: Identifier, Attempt
       - Simpler table, no expansion needed
       - If empty, show "No issues in retry queue"

       **e) Aggregate Metrics:**
       - Cards/summary row: Total Dispatched, Completed, Failed, Total Tokens
       - Use simple styled divs with Tailwind (bg-gray-800, rounded, p-4, etc.)

    4. SSE Integration:
       - On mount, create `new EventSource("/api/v1/events")`
       - On message, parse JSON, check event type
       - For dispatch/reconcile/stall/orch_retry events, trigger a re-fetch of `/api/v1/state` (debounce to avoid flood — use a 1-second throttle)
       - On unmount (component cleanup), close EventSource

    5. Error Handling:
       - If `/api/v1/state` returns 503, show "Orchestrator not configured" message instead of the dashboard
       - If fetch fails (network), show a warning banner but keep last known state

    Style notes:
    - Match the existing dashboard's dark theme (bg-gray-900, text-white/gray patterns)
    - Use consistent card/table patterns from existing pages
    - Keep it functional — no animations per convention
