  <name>Task 1: Implement GitHub Issues adapter with polling, pagination, ETag, and write-back</name>
  <files>src/tracker/github.ts, test/unit/tracker-github.test.ts</files>
  <behavior>
    - fetchCandidateIssues: fetches issues from GET /repos/{owner}/{repo}/issues with state=open, per_page=100, labels filter, since parameter for delta polling. Paginates via Link header. Filters out PRs (items with pull_request key). Normalizes to TrackerIssue[]. Updates lastUpdatedAt for delta polling. Uses ETag for conditional requests (sends If-None-Match, returns cached results on 304).
    - fetchIssueStatesByIds: fetches individual issues by number via GET /repos/{owner}/{repo}/issues/{number}, returns Map of id to state string
    - fetchIssuesByStates: fetches issues filtering by state parameter, normalizes results
    - postComment: POST /repos/{owner}/{repo}/issues/{number}/comments with body
    - updateState: PATCH /repos/{owner}/{repo}/issues/{number} with state field
    - updateLabels: POST to add labels, DELETE to remove individual labels
    - Rate limit: reads X-RateLimit-Remaining from response headers, stores rate limit info. When remaining drops below 100, logs warning. When 0, throws with reset time.
    - Normalization: id=String(issue.id), identifier="#"+issue.number, title, description=body||"", state, priority=null (or from label like "priority:high"), labels=labels[].name, assignees=assignees[].login, url=html_url, created_at, updated_at, blocked_by=[], metadata={reactions, pull_request}
  </behavior>
  <action>
    Create `src/tracker/github.ts`:

    1. **Internal state:** Store resolved token, owner, repo (split from config.repo "owner/repo"), labels filter, active_states, terminal_states, label config (in_progress_label, done_label). Also: lastETag (string|null), lastUpdatedAt (string|null), cachedIssues (TrackerIssue[]), rateLimitRemaining (number), rateLimitReset (number).

    2. **Helper: `githubFetch(path, options?)`** — Builds full URL `https://api.github.com${path}`, adds headers (Authorization: token {token}, Accept: application/vnd.github+json, User-Agent: forgectl). Reads rate limit headers from response and updates internal state. If response is 403 and rateLimitRemaining===0, throw with reset time. Returns response object.

    3. **Helper: `parseLinkHeader(header)`** — Parses Link header to extract rel="next" URL. Simple regex: `/<([^>]+)>;\s*rel="next"/`. Returns URL string or null.

    4. **Helper: `normalizeIssue(ghIssue)`** — Maps GitHub issue JSON to TrackerIssue. Filter logic: skip if `ghIssue.pull_request` is defined. Priority extraction: check labels for pattern like "priority:X" or "P0"/"P1" etc., otherwise null.

    5. **`fetchCandidateIssues()`** — Build query params: state from active_states[0] (GitHub only supports one state value), per_page=100, sort=updated, direction=desc. Add labels param if configured. Add since=lastUpdatedAt if set. Send If-None-Match: lastETag if set. On 304: return cachedIssues. On 200: store new ETag, paginate via Link header, normalize all issues, update lastUpdatedAt to max updated_at, cache results.

    6. **`fetchIssueStatesByIds(ids)`** — ids are identifier strings like "#42". Parse number, fetch each via GET /repos/{owner}/{repo}/issues/{number}. Return Map of id to state.

    7. **`fetchIssuesByStates(states)`** — For each state in states, fetch issues with that state param. Combine and normalize.

    8. **Write operations:** postComment, updateState, updateLabels — straightforward REST calls using githubFetch.

    9. **Factory:** Export `createGitHubAdapter(config: TrackerConfig): TrackerAdapter` — validates config.repo is present and contains "/", calls resolveToken(config.token), splits owner/repo, constructs and returns the adapter object.

    - Test updateLabels: add + remove calls

    - Test normalization: all fields mapped correctly, missing body defaults to ""
    - Test factory: throws on missing repo, throws on invalid repo format (no "/")
    - Set process.env.TEST_GH_TOKEN = "test-token" in beforeEach for token resolution
