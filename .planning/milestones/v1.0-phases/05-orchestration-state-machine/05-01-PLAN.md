    - classifyFailure("timeout") returns "error"
    - scheduleRetry stores timer handle in state.retryTimers
    - cancelRetry clears timer and removes from retryTimers map
    - clearAllRetries clears all timers
    - OrchestratorConfigSchema parses with defaults (enabled: false, max_concurrent_agents: 3, poll_interval_ms: 30000, stall_timeout_ms: 600000, max_retries: 5, max_retry_backoff_ms: 300000, drain_timeout_ms: 30000, continuation_delay_ms: 1000)
  </behavior>
  <action>
1. Create `src/orchestrator/retry.ts` with:
   - `calculateBackoff(attempt: number, maxBackoffMs: number): number` — returns `Math.min(10000 * Math.pow(2, attempt - 1), maxBackoffMs)`
   - `classifyFailure(status: AgentStatus): "continuation" | "error"` — "completed" maps to "continuation", everything else to "error"
   - `scheduleRetry(issueId, delayMs, callback, state)` — cancels existing timer for issueId if any, creates new setTimeout, stores handle in state.retryTimers
   - `cancelRetry(issueId, state)` — clearTimeout + delete from retryTimers
   - `clearAllRetries(state)` — iterate retryTimers, clearTimeout each, then clear the Map
   - Import AgentStatus from `../agent/session.js`, OrchestratorState from `./state.js`

   - poll_interval_ms: z.number().int().positive().default(30000)
   - stall_timeout_ms: z.number().int().positive().default(600000)

   - drain_timeout_ms: z.number().int().positive().default(30000)
   - continuation_delay_ms: z.number().int().min(0).default(1000)
   - in_progress_label: z.string().default("in-progress")

   Add `orchestrator: OrchestratorConfigSchema.default({})` to ConfigSchema.
   Export `OrchestratorConfig` type.
