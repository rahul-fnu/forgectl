## Cross-Phase Integration Findings

### Critical: SubIssueCache Not Threaded Through Orchestrator Composition Layer

All sub-issue runtime features (Phases 25 + 28) are fully implemented and unit-tested in their respective modules. However, `src/orchestrator/index.ts` — the composition root — does not thread `SubIssueCache` through to the three consumers:

| Location | Missing Wiring | Affected Requirements |
|----------|---------------|----------------------|
| `Orchestrator.start()` → `this.deps` | `subIssueCache` not in TickDeps | SUBISSUE-03 |
| `Orchestrator.dispatchIssue()` → `dispatchIssueImpl` | `subIssueCache` not passed as argument | SUBISSUE-05, SUBISSUE-06 |
| `server.ts` → `registerWebhookHandlers` | `subIssueCache` not in WebhookDeps | SUBISSUE-04 |

**Root cause:** All three gaps are in the same file (`src/orchestrator/index.ts`) and the daemon server (`src/daemon/server.ts`). The individual phase modules correctly accept optional `subIssueCache` parameters and guard against `undefined`, but the composition layer never provides the value.

**Impact:** The entire sub-issue runtime path (dependency-aware dispatch, progress rollup, auto-close) is structurally present but never executes. Features work in unit tests but not at runtime.
